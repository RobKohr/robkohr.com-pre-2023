emacs recursive find&grep in project directory

I do a lot of work in a set of directories in a project in emacs. The project has a huge number of files, and a lot of debugging involves tracking down little snippets of code or finding files.

To help with this, there are two really useful commands:

  • M-x grep-find - this will recursively find text in files in sub directories of the current directory
  • M-x find-name-dired - this will find files that contain a string of text in a specified directory (defaults to current).

grep-find is useful, especially since it provides links to the line numbers in the files that you can quickly jump to with a click or by pressing return on them. The down side, it is stuck to the current directory. If you want it to start at a custom directory, you can do so in your .emacs file.

(custom-set-variables                                         
’(grep-find-command “find ~/SOME_DIRECTORY ~/SOME_DIRECTORY_2 ~/SOME_DIRECTORY_N -type f -print0 | xar\gs -0 -e grep -nH -e ”)
)

You should only have one reference to custom-set-variables in your .emacs, if you already have one, just add the variable to that.

You can use as many directories as you like for searching.

Keyboard shortcuts are also nice, so I added these mappings:

(global-set-key (kbd “C-h”) ‘grep-find)
(global-set-key (kbd “C-R”) 'find-name-dired)

My only wish is that there was a way to specify the default starting directory for find-name-dired. Does anyone know what variable to set for this?
